# Guidelines

General AI collaboration guidelines for this workspace. These files are
reference material — not task boards, not operational instructions.

Read these when designing prompts, structuring agent communication, or
deciding when to use local models vs. core agents.

## Files

| File | Purpose |
|---|---|
| `llm-communication-rules.md` | When and how agents should communicate |
| `phatic-suppression.md` | Eliminating low-value agent messages |
| `laziness-ladder.md` | Choosing the cheapest capable agent for a task |
| `prompt-skeleton.md` | Reusable prompt structure for scoped agent tasks |
| `schema-pin.md` | Keeping agent output schemas stable across sessions |
| `context-pruning.md` | Trimming context windows before they degrade quality |
| `agent-file-watch-mode-outline.md` | Outline for file-watch-based agent triggers |
| `ai-improvement-scout-protocol.md` | Protocol for auditing and suggesting improvements |
| `map-local-model-roles-5-models-and-aider.md` | Which local models to use for which tasks |

See also: `MAP_System/notes/local-model-helper-guide.md` for the canonical
MAP-integrated version of local model policy.
