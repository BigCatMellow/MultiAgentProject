## Cleanup Applied

### MAP-specific gap review moved to MAP artifacts

Problem: `Guidelines/README.md` correctly says `Guidelines/` is universal, but
the folder also contained `Guidelines/MAP_repo_systems_gap_review.md`, a
MAP-specific audit/report. That contradicted the routing contract and matched
the operator's question about whether some Guidelines content should be inside
MAP.

Applied cleanup:

- Copied the canonical report to
  `MAP_System/artifacts/reports/MAP-repo-systems-gap-review.md`.
- Replaced `Guidelines/MAP_repo_systems_gap_review.md` with a short
  compatibility pointer to the canonical MAP artifact.
- Updated `Guidelines/README.md` with a compatibility-pointer section.
- Updated trusted live references in `MAP_System/shared/current-state.md` and
  `MAP_System/shared/decisions.md` to use the canonical MAP artifact path.

Historical review/release records and the 11 system docs still cite the old
`Guidelines/` path in places. That path now resolves to the pointer, so this
preserves history without broad churn.

## Cleanup Deferred

### `MAP_System/shared/DECISIONS.md`

`MAP_System/shared/DECISIONS.md` is an auto-generated decision index, while
`MAP_System/shared/decisions.md` is the canonical decision log. The uppercase
filename looks duplicate-prone, but it is generated by `validate_decisions.py`
and should not be renamed during this cleanup without first updating the
validator/output contract.

### Artifact subfolders

`MAP_System/artifacts/` has several historical subfolders (`drafts/`, `final/`,
`code/`, `command-center-ui/`) that are sparse but documented in
`artifacts/README.md`. They should remain unless a retention/archival task
decides to consolidate old artifact classes.

### Runtime/cache directories

`.venv/`, `.locks/`, `__pycache__/`, and `map.db` remain live runtime/tooling
state. They are not cleanup targets for a folder-structure pass.

## Result

The structure is now clearer in one material way:

- Universal guidance stays in `Guidelines/`.
- MAP-specific audit/report content has a canonical location under
  `MAP_System/artifacts/reports/`.
- Old links keep resolving through a compatibility pointer.

No destructive moves or deletes were performed.
