# Archive

Archive stores historical memory that should remain available but should not
live in the active project brain.

Use archive for:

- compacted event summaries;
- old detailed narratives that have been reduced into a current-state summary;
- superseded planning material;
- raw extracts preserved for auditability.

Do not use archive for:

- current requirements;
- active decisions;
- unresolved questions;
- task records that the runner still needs;
- handoffs needed for current work.

Active agents should read archive only when a current task, compaction summary,
or decision points to a specific archived file.

## Contents

- `compactions/` — brain-compaction summaries per
  `notes/brain-compaction-guide.md`. One file per compaction pass, named
  `compaction-YYYY-MM-DD-tasks-NNN-NNN.md`. First entry: 2026-07-14
  (TASK-193, tasks 147–192).
- `general-purpose-multi-agent-project-system-v2.md` — superseded early
  system design, preserved for provenance.
