# Notes

Notes are operating guidance and runbooks. They explain how to use MAP, but they
are secondary to `AGENTS.md`, `shared/current-state.md`, and current task files.

## Current Notes

- `operations-runbook.md` - status, validation, runner, and recovery commands.
- `task-authoring-guide.md` - how to create and repair task records.
- `task-metadata-repair-plan.md` - focused plan for repairing current task metadata gaps.
- `state-machine-guardrails.md` - READY gate and task-state transition guardrails.
- `architect-agent-guide.md` - Architect/Shaper role for turning ideas into claimable tasks.
- `review-guide.md` - review verdicts, severities, and artifact format.
- `helper-agent-guide.md` - helper-agent setup, notes, and stop conditions.
- `local-model-helper-guide.md` - local Ollama/Aider helper policy and boundaries.
- `communication-guide.md` - routing rules for core agents, helpers, inbox notes, and handoffs.
- `communication-architecture.md` - thread IDs, message IDs, promotion rules, and cross-references.
- `documentation-style-guide.md` - structured writing rules for all MAP files.
- `context-routing-guide.md` - which MAP files to read together for common situations.
- `brain-compaction-guide.md` - how to keep active memory lean while preserving history.
- `brain-organization-guide.md` - reusable guide for organizing MAP-style project memory.
- `command-center-later.md` - deferred hardening items for the next command-center session.
- `git-setup.md` - Git wrapper and local repository notes.
- `how-to-work-with-codex-and-claude.md` - collaboration loop.

## What Belongs Here

- Runbooks.
- Stable operator procedures.
- Practical guidance that applies across many tasks.

## What Does Not Belong Here

- Task-specific work products.
- Review records.
- Current project truth that belongs in `shared/`.
